# 博客部署文档（新机器）

本博客基于 **Hugo Extended** + **PaperMod 主题**，托管在 GitHub，自定义域名 `https://fan-pengfei.top/`，通过 GitHub Actions 自动部署。

> ⚠️ **最重要的一点**：主题 PaperMod 是用 **git submodule** 引入的。新机器克隆仓库时**必须带 `--recurse-submodules`**，否则 `themes/PaperMod` 目录是空的，构建会失败。

## 环境要求

| 工具 | 版本要求 | 说明 |
|------|----------|------|
| Git | 任意较新版本 | 克隆仓库、提交代码 |
| Hugo | **Extended v0.162.1**（或更高） | 必须是 **extended** 版，PaperMod 依赖它编译 SCSS |

> 验证 Hugo 是否为 extended 版：运行 `hugo version`，输出里要带 `+extended` 字样。

## Windows 部署

### 1. 安装 Git

方式 A（推荐，命令行）：用 winget
```powershell
winget install Git.Git
```

方式 B：去 https://git-scm.com/download/win 下载安装包，一路下一步。

安装后**重开一个终端**，验证：
```powershell
git --version
```

### 2. 安装 Hugo Extended

方式 A（推荐）：winget
```powershell
winget install Hugo.Hugo.Extended
```

方式 B：scoop
```powershell
scoop install hugo-extended
```

方式 C：手动下载二进制
1. 打开 https://github.com/gohugoio/hugo/releases
2. 下载 `hugo_extended_0.162.1_windows-amd64.zip`
3. 解压，把里面的 `hugo.exe` 放到某个目录（如 `C:\hugo\`）
4. 把该目录加入系统环境变量 `Path`

验证（**装完重开终端**）：
```powershell
hugo version
```
应看到类似 `hugo v0.162.1 ... +extended windows/amd64`。

接着跳到 [克隆仓库](#克隆仓库两个系统通用)。

## Ubuntu 部署

### 1. 安装 Git

```bash
sudo apt update
sudo apt install -y git
git --version
```

### 2. 安装 Hugo Extended

> ⚠️ 不要直接 `apt install hugo`，仓库里的版本通常太旧。请用下面的方式装指定版本。

方式 A（推荐）：下载官方 .deb 包
```bash
HUGO_VERSION=0.162.1
wget -O /tmp/hugo.deb \
  https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.deb
sudo dpkg -i /tmp/hugo.deb
```

方式 B：snap
```bash
sudo snap install hugo
```

方式 C：免 sudo，装到用户目录（适合无管理员权限）
```bash
HUGO_VERSION=0.162.1
mkdir -p ~/.local/bin
curl -sL -o /tmp/hugo.tar.gz \
  https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.tar.gz
tar -xzf /tmp/hugo.tar.gz -C /tmp hugo
mv /tmp/hugo ~/.local/bin/hugo
chmod +x ~/.local/bin/hugo
# 确保 ~/.local/bin 在 PATH 中（若 hugo 命令找不到，把下面这行加进 ~/.bashrc 后重开终端）
# export PATH="$HOME/.local/bin:$PATH"
```

验证：
```bash
hugo version
```
应看到 `hugo v0.162.1 ... +extended linux/amd64`。

接着看 [克隆仓库](#克隆仓库两个系统通用)。

## 克隆仓库（两个系统通用）

### 第一步：配置 GitHub 访问

仓库地址：
- SSH：`git@github.com:fan-pengfei/blog.git`
- HTTPS：`https://github.com/fan-pengfei/blog.git`

新机器若用 SSH，需要先在这台机器生成 SSH key 并加到 GitHub：
```bash
ssh-keygen -t ed25519 -C "你的邮箱"
# 一路回车，然后查看公钥：
cat ~/.ssh/id_ed25519.pub
```
把输出的公钥内容粘到 GitHub → Settings → SSH and GPG keys → New SSH key。

> 嫌麻烦也可以直接用 HTTPS 地址克隆，推送时按提示登录（需用 Personal Access Token 作为密码）。

### 第二步：克隆（务必带 submodule）

```bash
git clone --recurse-submodules git@github.com:fan-pengfei/blog.git
cd blog
```

> 如果你已经克隆了但没带 `--recurse-submodules`，补救命令：
> ```bash
> git submodule update --init --recursive
> ```

### 第三步：确认主题已拉下来

```bash
ls themes/PaperMod
```
能看到 `layouts`、`assets` 等文件夹就对了。如果是空的，回去执行上面的 `git submodule update`。

## 本地预览

在项目根目录运行：

```bash
hugo server -D
```

- `-D` 表示同时渲染草稿（draft: true 的文章）
- 浏览器打开 **http://localhost:1313/**
- 带热重载：改文件保存后页面自动刷新
- 按 `Ctrl + C` 停止

> 提示：本地预览时 Hugo 会自动用 `localhost` 地址，不受配置里 `baseURL` 影响，无需改动。

## 写文章与发布

### 新建文章

```bash
hugo new content posts/我的新文章.md
```

文件会生成在 `content/posts/`，打开它，编辑顶部的 front matter 和正文：

```markdown
---
title: "文章标题"
date: 2026-06-05T10:00:00+08:00
draft: false          # 改成 false 才会正式发布
tags: ["标签1", "标签2"]
categories: ["分类"]
summary: "文章摘要"
---

正文内容写在这里……
```

### 本地确认

```bash
hugo server -D
```
浏览器看效果满意后再发布。

### 发布上线

提交并推送，GitHub Actions 会自动构建部署，几十秒后线上生效：

```bash
git add .
git commit -m "post: 新文章标题"
git push
```

推送后可在仓库 **Actions** 标签页看部署进度，全绿即上线。访问 https://fan-pengfei.top/ 查看。

## 常见问题

**Q：构建报错找不到主题 / 页面没样式？**
A：主题 submodule 没拉下来。运行 `git submodule update --init --recursive`。

**Q：`hugo version` 不带 `+extended`？**
A：装错成了普通版。卸载后按上面步骤重装 **extended** 版，否则 PaperMod 的 SCSS 无法编译。

**Q：`hugo` 命令找不到（command not found）？**
A：Hugo 没装好或不在 PATH。Windows 检查环境变量 Path；Ubuntu 用方式 C 安装的需确保 `~/.local/bin` 在 PATH 中。

**Q：本地预览端口被占用？**
A：换端口运行 `hugo server -D --port 1314`。

**Q：推送后网站没更新？**
A：去仓库 Actions 看那次工作流是否成功（绿勾）。失败的话点开看报错。注意 GitHub Pages 有 CDN 缓存，偶尔需多等几分钟并强制刷新（Ctrl+F5）。

**Q：换了机器后想改站点配置？**
A：编辑根目录 `hugo.toml`（标题、菜单、作者、社交链接等），改完同样 `git push` 发布。
